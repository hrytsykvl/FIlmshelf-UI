# FilmShelf

FilmShelf — це Angular-фронтенд для платформи відстеження та відкриття фільмів. Користувачі переглядають каталог, оцінюють і рецензують стрічки, формують списки перегляду та досліджують персоналізовані рекомендації, створені кількома різними алгоритмами — від класичної колаборативної фільтрації до великих мовних моделей.

Інтерфейс взаємодіє з окремим FilmShelf API (.NET-сервіс, розгорнутий в Azure Container Apps) та отримує оновлення в реальному часі через SignalR.

## Основні можливості

- **Фільми** — перегляд каталогу з пагінацією, детальні сторінки фільмів (акторський склад, середня оцінка), сторінки акторів та пошук по бібліотеці.
- **Рецензії та оцінки** — створення, редагування та видалення рецензій з числовою оцінкою, відповіді під будь-якою рецензією та перегляд усіх написаних вами рецензій з окремого розділу профілю. Кожен фільм має агреговану середню оцінку, яка використовується по всьому інтерфейсу.
- **Списки перегляду** — у кожного користувача є типовий список перегляду плюс будь-яка кількість користувацьких списків. Фільми можна додавати/видаляти зі списку одним кліком, а інтерфейс відстежує належність до списку на всіх сторінках, тому стан "у списку перегляду" завжди синхронізований.
- **Рекомендації** — єдина стрічка рекомендацій надає шість різних стратегій рекомендацій під одним перемикачем; див. розділ [Рекомендації](#рекомендації) нижче.
- **Сповіщення в реальному часі** — відповіді на рецензії та інші події щодо фільмів надходять через хаб SignalR, з лічильником непрочитаних сповіщень та елементами керування "прочитано/видалити" для кожного запису.
- **Автентифікація** — реєстрація email/пароль з JWT access + refresh токенами, вхід через Google OAuth, а також процеси "забув/скинути пароль".

## Рекомендації

Стрічка рекомендацій за адресою `/movies/recommendation-feed` дозволяє користувачам перемикатися між двома режимами перегляду та шістьма методами рекомендацій, забезпеченими API.

### Режими перегляду

- **Сітка** — швидке, щільне розташування рекомендованих постерів. Використовується для класичних/статистичних методів.
- **Детальний** — карткове розташування, яке на додачу до фільму показує **оцінку** моделі та **причину** для кожного фільму, що пояснює, чому його було рекомендовано. Використовується для методів на основі LLM.

### Методи рекомендацій

| Метод | Режим | Що робить |
| --- | --- | --- |
| **Matrix Factorization (ML.NET)** (`ml`) | Сітка | Модель матричної факторизації, навчена на оцінках користувачів/фільмів. За замовчуванням для режиму сітки. |
| **Content-based** (`content`) | Сітка | Ранжує кандидатів за подібністю жанрів, режисера та акторського складу до фільмів, які користувачу вже подобаються. |
| **User Collaborative Filtering** (`user-cf`) | Сітка | Кореляція Пірсона між користувачами для пошуку "сусідів" з подібними смаками. |
| **Azure Embeddings** (`embedding`) | Сітка | Ембединги Azure OpenAI у поєднанні з векторною подібністю Azure AI Search. |
| **Claude (AI reasoning)** (`llm`, провайдер `claude`) | Детальний | Claude генерує рекомендації та природномовне пояснення для кожного вибору. За замовчуванням для детального режиму. |
| **GPT (OpenAI)** (`gpt`, провайдер `openai`) | Детальний | GPT-варіант того ж потоку з поясненням для кожного фільму. |

Зміна режиму автоматично обирає розумний метод за замовчуванням для цього режиму; зміна методу повторно завантажує стрічку. Результати звіряються зі списками перегляду користувача, тому елемент "додати до списку перегляду" відображає актуальну належність.

Інші поверхні рекомендацій у застосунку:
- `/movies/popular` — стрічка популярного зараз, що працює на ендпоінті фільмів `?filter=popular`.

## Технологічний стек

- **Angular 18** (standalone-компоненти, новий синтаксис керування потоком у шаблонах, SCSS)
- **Angular Material 17** + **Angular CDK** + **ng-heroicons** для UI-примітивів
- **RxJS** для асинхронного потоку даних
- **@auth0/angular-jwt** для роботи з JWT, з HTTP-інтерсептором, який додає токени та оновлює їх за потреби
- **@abacritt/angularx-social-login** для входу через Google
- **@microsoft/signalr** для сповіщень у реальному часі
- **ngx-pagination** для списків з пагінацією
- **dotenv-webpack** (через `@angular-builders/custom-webpack`), щоб `process.env['…']` розв'язувалося під час збірки з файлу `.env`

## Структура проєкту

```
FilmShelf/
├── src/app/
│   ├── movies/, movie/, movie-details/, movie-content/   # каталог + сторінки фільмів
│   ├── popular-movies/, recommended-movies/,
│   │   recommendation-feed/                               # поверхні рекомендацій
│   ├── review/, reviews/, user-reviews/                  # рецензії та оцінки
│   ├── watchlist/, custom-lists/, custom-list/,
│   │   custom-list-details/, create-list/                # списки перегляду та користувацькі списки
│   ├── actor/, actor-details/                            # сторінки акторського складу
│   ├── login/, register/, forgot-password/, reset-password/  # процеси автентифікації
│   ├── navbar/, search-bar/, pagination/, notification/  # спільний UI
│   ├── services/      # обгортки HttpClient для кожного ресурсу + клієнт SignalR-хабу
│   ├── models/        # DTO запитів/відповідей
│   ├── interceptors/  # token interceptor
│   ├── helpers/       # допоміжні модулі для авторизації + списків перегляду
│   ├── validators/    # валідатори реактивних форм
│   └── constants/     # URL-и API, URL-и хабу, ключі сховища
├── Dockerfile         # багатоетапна збірка → nginx:alpine
├── nginx.conf         # слухає 8080 для Azure Container Apps
└── angular.json
```

### Використовувана поверхня API

Базова URL — `${API_BASE_URL}`; SPA звертається до:

- `/account/*` — register, token, refresh-token, logout, google-login, forgot/reset password, reviews
- `/movie`, `/movie/{id}`, `/movie/search`, `/movie/{id}/reviews`
- `/movie/recommendations?method=…` та `/movie/recommendations/llm?provider=…`
- `/watchlist`, `/watchlist/{id}`, `/watchlist/{id}/movies/{movieId}`, `/watchlist/movies`
- `/review`, `/review/{id}`, `/review/{id}/responses`, `/review/responses/{id}`
- `/actor/*`
- `/notification`, `/notification/unread`, `/notification/{id}/read`

Плюс SignalR-хаб для подій `ReceiveNotification` та `ReceiveMovieNotification`.

## Початок роботи

Передумови: Node 20+ та запущений FilmShelf API.

```bash
cd FilmShelf
npm ci
```

Створіть файл `.env` поруч з `package.json` зі значеннями, які бандлер інжектує:

```env
API_BASE_URL=https://<your-api-host>/api
BASE_URL=https://<your-api-host>
APP_BASE_URL=http://localhost:4200
GOOGLE_CLIENT_ID=<your-google-oauth-client-id>
```

Далі:

```bash
npm start         # дев-сервер на http://localhost:4200
npm run build     # продакшн-збірка → dist/film-shelf
npm test          # Karma + Jasmine
```

## Docker

Багатоетапний Dockerfile збирає SPA за допомогою Node і обслуговує її через nginx на порту 8080 (типовий порт для Azure Container Apps):

```bash
docker build \
  --build-arg API_BASE_URL=https://<your-api>/api \
  --build-arg BASE_URL=https://<your-api> \
  --build-arg APP_BASE_URL=https://<your-ui> \
  --build-arg GOOGLE_CLIENT_ID=<client-id> \
  -t filmshelf-ui ./FilmShelf

docker run --rm -p 8080:8080 filmshelf-ui
```

`ARG`-и часу збірки записуються у файл `.env`, який `dotenv-webpack` читає під час `ng build`, тому значення вшиваються у вироблений бандл.

## Розгортання

Включений Dockerfile та `nginx.conf` націлені на **Azure Container Apps**. Репозиторій вже містить конфігурацію `azure-pipelines`/CI, яка збирає образ і пушить його; деталі — див. CI-воркфлоу.
